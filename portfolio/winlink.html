动态组件设计 － 幸运转盘

  • ABOUT THE PROJECT - 项目加塞心更塞

    互动场景是微商城的模块功能，通过在页面上添加组件并编辑生成h5页面。用于活动页、展示页等的制作，致力于为客户打造简洁高效的h5编辑器。

    临近春节，大多数客户要求制作节日相关的活动抽奖页。有时会打断项目的进度。并且通常从设计到制作完成，需要2～3天左右的时间，客户不满意的话还需要修改。在现有云平台里的静态页面制作器的基础上，有了动态组件的制作想法。

    身为前端开发，也参与了部分设计。这次我们首先单独设计了幸运轮盘这一组件。

局限

上图是现有界面的样式，编辑组件的窗口由于位置拥挤，但组件之间无上下文关系，可以独立设置为弹窗。所以，如何解决在弹窗状态下将轮盘的内容清晰地进行编辑成了一个难题。

问题

在网上找了一些微信第三发开发平台上的 "幸运转盘" 插件，发现大多有以下问题：

• 设置机制让用户难以清楚

在网上的大多数转盘制作插件中，基本是通过添加奖项，按照奖项的多少添加转盘瓣，不能自己定义瓣数，奖项的位置也不能自定义。

• 无法预知每一步带来的效果

设置后需要按下预览键后，弹跳出新窗口显示最终效果，在出错的情况下还需重新设置后再打开新的窗口浏览，让用户倍感疲累。

设计预想

因为只能在弹窗中设置轮盘的相关信息，又希望可以较直观的编辑轮盘每个瓣的信息。所以首先将编辑区域划分为轮盘区和编辑区。
按照这个想法设计出了初稿，但设计完后再仔细看就有问题了：

• 页面中的功能分布不合理，难以对进行后续的功能拓展。
• 虽然可以点击任意一个瓣进行编辑，但只有在退出编辑后才能知道编辑后的效果，且容易忘记哪个瓣已编辑过。

再设计

再次的设计，针对于以上的问题，做出了以下的改进：

• 预览区可以直接预览最终的效果，点击任意一瓣即可边编辑边看到效果。
• 将编辑区的功能拆分成三个模块，让用户可以专注于当前模块功能的编辑，不受其它模块功能的影响。

反馈

将设计放至我所参加的小密圈（交互设计小厨房）中获取反馈，hozin指出了几个改进点：

• tab的应用容易导致用户多次点击检查，增加点击检查负担。
• 轮盘瓣只有9瓣，可以做增值列表，以文字来叙述轮盘的情况，减少直接使用轮盘做交互的问题。

看了意见后，想了想：

• 不使用tab，直接罗列选项，减少切换tab的点击次数。
• 轮盘做交互的话有以下几点问题：
1、不点击轮盘瓣设置模块时，任何在轮盘上的点击操作都无法进行瓣内容编辑。
2、在轮盘瓣设置模块编辑时，用户要在轮盘和内容编辑区视线来回切换，比较曲折麻烦。

再再设计

根据建议改进，去除轮盘预览，编辑轮盘改为增值列表，把每个模块内容平铺出来，让用户一目了然。

总结

"解决问题的过程就是不断把大问题拆分成小问题来解决。"这是刚才是接触这个项目时组长对我说的一句话，这句话现在看来也适用于设计， 将功能类似的类目归类为同一项，既能够集中于编辑，又能在修改查找时按照大类快速查找。
考虑清楚问题的限制性让我明白了要在合理的范围内才能更好的解决问题。
